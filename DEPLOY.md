# AcadCode — Deployment Guide

## What's New in v2.1 (AI Daily Questions)

- Daily questions are now **AI-generated by Claude** at midnight PHT (Asia/Manila, UTC+8)
- Each day has a **subject** based on day-of-week rotation:
  - Sunday → General Knowledge
  - Monday → Science
  - Tuesday → Mathematics
  - Wednesday → Computer Science
  - Thursday → History & Social Studies
  - Friday → English & Literature
  - Saturday → Arts & Culture
- Difficulty is mixed per day: elementary → college level
- A new `questions` table stores each day's set

---

## Prerequisites

- Vercel account (free tier works)
- Neon Postgres database (free tier works — connect via Vercel integrations)
- Anthropic API key — get one at https://console.anthropic.com

---

## 1. Create the Database

Run `db/schema.sql` against your Neon database once:

    psql "$DATABASE_URL" -f db/schema.sql

Or paste the contents into Vercel Storage → your DB → Query tab.

This creates: `users`, `sessions`, `scores`, and the new `questions` table.

---

## 2. Set Environment Variables in Vercel

Go to your project → Settings → Environment Variables and add:

    DATABASE_URL       = Your Neon connection string (pooled)
    GEMINI_API_KEY     = your key from ai.google.dev (free, no credit card)
    CRON_SECRET        = Any random string (openssl rand -hex 32)

---

## 3. Deploy

    vercel deploy --prod

Vercel will automatically register the cron job from vercel.json:
- Schedule: 0 16 * * * = 16:00 UTC = 00:00 midnight PHT
- Endpoint: POST /api/generate-questions

NOTE: Vercel Cron requires the Pro plan or higher. On the free Hobby
plan, trigger question generation manually (see step 4).

---

## 4. Generate Today's Questions (First Run / Manual Trigger)

After deploying, seed today's questions:

    curl -X POST https://your-app.vercel.app/api/generate-questions \
      -H "Authorization: Bearer YOUR_CRON_SECRET"

The endpoint is idempotent — if today already has 10 questions it does nothing.

---

## 5. How It Works

    midnight PHT
        |
        v
    Vercel Cron --> POST /api/generate-questions
        |
        |-- Determines today's subject (day-of-week lookup)
        |-- Calls Claude API (claude-sonnet-4-20250514)
        |     '-- Prompt: "Generate 10 MCQs about {subject}, mixed difficulty, return JSON"
        |-- Parses JSON response (10 question objects)
        '-- Saves to `questions` table with today's PHT date

    User loads page
        |
        v
    GET /api/questions --> returns today's rows from DB
        |
        '-- Frontend shows subject badge + loads quiz

---

## 6. Fallback Behavior

- If the cron fails, users see: "No questions available yet. Check back shortly!"
- Manually trigger /api/generate-questions to recover
- Once generated, questions are served from DB — no AI calls during quiz play

---

## 7. Local Development

    npm install
    vercel env pull .env.local
    # Manually add ANTHROPIC_API_KEY and CRON_SECRET to .env.local
    vercel dev

Test generation locally:

    curl -X POST http://localhost:3000/api/generate-questions \
      -H "Authorization: Bearer YOUR_CRON_SECRET"

---

## API Endpoints Summary

    GET  /api/questions          (no auth)     Today's 10 questions
    POST /api/generate-questions (CRON_SECRET) Generate via AI
    GET  /api/status             (Bearer)      Has user played today?
    POST /api/submit             (Bearer)      Submit score
    GET  /api/me                 (Bearer)      User profile
    GET  /api/leaderboard        (no auth)     Global leaderboard
    POST /api/login              (no auth)     Login
    POST /api/register           (no auth)     Register
    POST /api/logout             (Bearer)      Logout
